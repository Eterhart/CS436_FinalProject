## ภาพรวมโครงการ
แอปพลิเคชันบนเว็บสำหรับสร้างรหัสผ่านและข้อความรหัสผ่านแบบสุ่ม ที่ช่วยให้ผู้ใช้สามารถสร้างรหัสผ่านที่แข็งแกร่งและปลอดภัยด้วยตัวเลือกที่ปรับแต่งได้

## วัตถุประสงค์ของโครงการ
1. จัดหาเครื่องมือสร้างรหัสผ่านที่ปลอดภัยฝั่งไคลเอนต์ให้กับผู้ใช้
2. เสนอตัวเลือกการปรับแต่งที่ยืดหยุ่นสำหรับความซับซ้อนของรหัสผ่าน
3. ให้ความรู้แก่ผู้ใช้เกี่ยวกับความปลอดภัยของรหัสผ่านผ่านตัวบ่งชี้ความแข็งแกร่งแบบเรียลไทม์
4. ช่วยให้ผู้ใช้สามารถสร้างทั้งรหัสผ่านแบบดั้งเดิมและข้อความรหัสผ่านที่จดจำง่าย
5. อนุญาตให้ผู้ใช้ติดตามประวัติการสร้างรหัสผ่านของตน

## ขอบเขตที่อยู่ในโครงการ (IN SCOPE)

### ฟีเจอร์หลัก
- **การสร้างรหัสผ่าน**
  - กำหนดความยาวรหัสผ่านได้ (4-128 ตัวอักษร)
  - เลือกประเภทตัวอักษร (ตัวพิมพ์ใหญ่, ตัวพิมพ์เล็ก, ตัวเลข, สัญลักษณ์)
  - กำหนดจำนวนขั้นต่ำของตัวเลขและสัญลักษณ์
  - ตัวเลือกหลีกเลี่ยงตัวอักษรที่คลุมเครือ
  - สร้างรหัสผ่านแบบเรียลไทม์

- **การสร้างข้อความรหัสผ่าน (Passphrase)**
  - สร้างข้อความรหัสผ่านจากคำศัพท์ (2-10 คำ)
  - ตัวเลือกตัวคั่นหลายแบบ (ช่องว่าง, ขีดกลาง, ขีดล่าง, จุด, ไม่มี)
  - ตัวเลือกการใช้ตัวพิมพ์ใหญ่-เล็ก (lowercase, uppercase, title case)
  - ตัวเลือกการรวมตัวเลข

- **ฟีเจอร์ด้านความปลอดภัย**
  - ตัวบ่งชี้ความแข็งแกร่งด้านความปลอดภัยแบบเรียลไทม์
  - การคำนวณเวลาโดยประมาณในการถอดรหัส
  - เคล็ดลับและแนวทางปฏิบัติที่ดีด้านความปลอดภัย
  - การสร้างฝั่งไคลเอนต์ (ไม่มีการส่งข้อมูลไปยังเซิร์ฟเวอร์)

- **ส่วนติดต่อผู้ใช้**
  - สลับโหมด (รหัสผ่าน/ข้อความรหัสผ่าน)
  - คัดลอกไปยังคลิปบอร์ดด้วยคลิกเดียว
  - การตอบสนองทางภาพสำหรับการคัดลอก
  - การออกแบบที่รองรับทุกอุปกรณ์
  - ติดตามประวัติรหัสผ่าน
  - โหลดค่าตั้งต้นใหม่จากรายการประวัติ

### การพัฒนาด้านเทคนิค
- แอปพลิเคชันหน้าเดียวที่ใช้ React
- TypeScript เพื่อความปลอดภัยของชนิดข้อมูล
- Tailwind CSS สำหรับการจัดรูปแบบ
- ระบบบิลด์ Vite
- Lucide React สำหรับไอคอน

## ขอบเขตที่ไม่อยู่ในโครงการ (OUT OF SCOPE)

### ฟีเจอร์ที่ไม่รวมอยู่
- ระบบยืนยันตัวตนหรือบัญชีผู้ใช้
- การจัดเก็บหรือซิงค์บนคลาวด์
- ฟังก์ชันเก็บรหัสผ่านหรือ vault
- การรวมกับส่วนขยายของเบราว์เซอร์
- การรองรับหลายภาษา
- การทดสอบความแข็งแกร่งของรหัสผ่านที่มีอยู่
- การสร้างรหัสผ่านจำนวนมากพร้อมกัน
- ฟังก์ชันส่งออก (CSV, JSON, ฯลฯ)
- การอัปโหลดรายการคำศัพท์แบบกำหนดเองสำหรับข้อความรหัสผ่าน
- การติดตามอายุหรือการหมดอายุของรหัสผ่าน
- การรวมกับโปรแกรมจัดการรหัสผ่าน
- API endpoints หรือบริการ backend
- แอปพลิเคชันมือถือแบบ native
- ฟังก์ชัน PWA แบบออฟไลน์
- การวิเคราะห์หรือติดตามการใช้งาน

### ข้อจำกัดด้านเทคนิค
- ไม่มีการประมวลผลฝั่งเซิร์ฟเวอร์
- ไม่มีการเก็บข้อมูลในฐานข้อมูล
- ไม่มีการเก็บรวบรวมข้อมูลผู้ใช้
- ไม่มีการยืนยันตัวตนจากบุคคลที่สาม
- ไม่มีการจัดการคีย์เข้ารหัส

## เกณฑ์ความสำเร็จ
1. ผู้ใช้สามารถสร้างรหัสผ่านด้วยตัวเลือกที่ปรับแต่งได้
2. ผู้ใช้สามารถสร้างข้อความรหัสผ่านที่ประกอบด้วยคำศัพท์
3. ความแข็งแกร่งด้านความปลอดภัยคำนวณและแสดงผลอย่างถูกต้อง
4. การคัดลอกไปยังคลิปบอร์ดทำงานได้ในเบราว์เซอร์หลักทั้งหมด
5. แอปพลิเคชันรองรับการใช้งานบนมือถือ แท็บเล็ต และเดสก์ท็อป
6. ประวัติเก็บรหัสผ่าน/ข้อความรหัสผ่านที่สร้างล่าสุด 10 รายการ
7. การสร้างทั้งหมดเกิดขึ้นฝั่งไคลเอนต์โดยไม่มีการเรียกเครือข่าย

## กลุ่มเป้าหมาย
- ผู้ใช้ทั่วไปที่ต้องการรหัสผ่านที่ปลอดภัย
- ผู้เชี่ยวชาญด้าน IT ที่ต้องการสร้างรหัสผ่านอย่างรวดเร็ว
- บุคคลที่ใส่ใจเรื่องความปลอดภัย
- ทุกคนที่กำลังสร้างบัญชีออนไลน์ใหม่

## ข้อจำกัด
- ต้องทำงานในเบราว์เซอร์ทั้งหมด
- ไม่มีการพึ่งพาภายนอกสำหรับตรรกะการสร้างรหัสผ่าน
- ต้องรักษาความเป็นส่วนตัวของผู้ใช้ (ไม่มีการส่งข้อมูล)
- ต้องทำงานแบบออฟไลน์เมื่อโหลดแล้ว

## สมมติฐาน
- ผู้ใช้มีเบราว์เซอร์สมัยใหม่ (Chrome, Firefox, Safari, Edge)
- ผู้ใช้เข้าใจแนวคิดพื้นฐานด้านความปลอดภัยของรหัสผ่าน
- ผู้ใช้เปิดใช้งาน JavaScript
- ผู้ใช้มีสิทธิ์เข้าถึงคลิปบอร์ด
